# NBA Player Performance — Static (GitHub Pages) build

A **fully static** version of the Flask app, designed to run on **GitHub Pages**.
It has no server and no database. Instead:

- **Stats & dashboard data** are pre-exported to JSON (`data/*.json`) at build time.
- **Predictions** are made by calling the Hugging Face Space
  [`aggtamv/nba_plusminus`](https://huggingface.co/spaces/aggtamv/nba_plusminus)
  **directly from the browser** via the Gradio JS client. The Space's CORS policy
  allows this.

```
GitHub Pages (static)                 Hugging Face Space (model API)
┌───────────────────────────┐        ┌────────────────────────────┐
│ index / stats / prediction │        │  aggtamv/nba_plusminus     │
│ Plotly.js + data/*.json ───┼─draw─▶ │                            │
│ @gradio/client ────────────┼─POST──▶│  /predict_from_csv → text  │
└───────────────────────────┘        └────────────────────────────┘
```

## Refreshing the data

The JSON in `data/` is a snapshot. Three ways to refresh it:

| Script | Scrapes | When |
|---|---|---|
| `export_data.py` | nothing (re-reads existing CSVs) | after you scrape manually |
| `refresh_daily.py` | latest match scores + boxscores | daily |
| `refresh_season.py` | yearly player totals + shooting averages | monthly / yearly |

Run any of them from the **project root**, e.g.:

```bash
python static_page/refresh_daily.py     # scrape latest games + re-export
python static_page/refresh_season.py    # scrape season stats + re-export
python static_page/export_data.py       # re-export only (no scraping)
```

### Automated schedule (no server needed)

Two scheduled **GitHub Actions** keep the site fresh — GitHub Pages itself has no
scheduler, but Actions (free CI in the same repo) runs on a cron, scrapes on
GitHub's servers, commits the new JSON, and redeploys Pages:

- [`.github/workflows/refresh-daily.yml`](../.github/workflows/refresh-daily.yml)
  — daily at 11:00 UTC (latest match). Installs headless Chrome for the Selenium fallback.
- [`.github/workflows/refresh-season.yml`](../.github/workflows/refresh-season.yml)
  — monthly on the 1st (season stats). Plain `requests`, no browser.

Both have a **"Run workflow"** button in the Actions tab for on-demand refreshes.
Cron times are **UTC**, and scheduled workflows auto-disable after 60 days of repo
inactivity (any push or manual run re-arms them).

> If a scrape is ever blocked by the source site from CI, the run still re-exports
> from the last-good CSVs (the scrape step is non-fatal), so the site never breaks.

## Local preview

Open with any static server from inside `static_page/` (needed because pages
`fetch()` the JSON — `file://` won't work):

```bash
cd static_page
python -m http.server 8000
# then visit http://localhost:8000
```

## Notes / limitations

- The Hugging Face Space **sleeps after inactivity**; the first prediction after
  idle can take ~30s while it wakes. The UI shows a "waking up" message.
- Predictions for the **Player** tab are limited to players who had ≥10 complete
  recent games at export time (see `data/player_names.json`). The **CSV Upload**
  tab accepts any valid 10×20 feature file (`sample/input_sample.csv`).
